我们先来讨论equals()，根据String.java的源码我们很容易可知equals()比较的是两个变量所指向的对象的值

但是当str1 == str2得到的结果为true，我们会觉得摸不着头脑，难道结果不应该是false吗，难道不是因为str1和str2在栈中分别new了两个新的对象，两个新的对象地址值不同，所以两个引用变量通过==运算结果才为false?

原因是：字符串常量池(String Pool)

2. 字符串常量池

  • 常量池：用于保存Java在编译期就已经确定的，已经编译的class文件中的一份数据。包括了类、方法、接口中的常量，也包括字符串常量，如String str1 = "hello";这种声明方式

  • 为了减少在JVM中创建的字符串的数量，字符串类维护了一个字符串常量池，字符串常量池是Java堆内存中一个特殊的存储区域

  • 当创建String对象时，JVM会先检查字符串常量池，如果这个字符串的常量值已经存在在池中了，就直接返回池中对象的引用，如果不在池中，就会实例化一个字符串并放入池中

所以根据字符串常量池的相关概念，当String str1 = "hello";，JVM便会在字符串常量池中搜索是否存在字符串”hello”，如果没有，便会实例化生成一个字符串”hello”存入池中。当String str2 = "hello";，JVM发现在字符串常量池中存在字符串”hello”，便会返回一个引用即字符串”hello”的内存地址0x100赋值给str2，所以str1 == str2的结果为true，因为它们指向的都是同一个字符串对象”hello”

3. intern() 方法

当调用intern()方法时，如果字符串常量池中已经包含一个等于此 String 对象的字符串，则返回池中的字符串引用。否则，将此 String 对象添加到池中，并且返回此 String 对象的引用
